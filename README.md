# Git Commit Time Analyzer

一个用于分析 Git 仓库提交时间的小工具，可以统计提交时间的分布、计算平均提交时间。

## 背景

深夜，因学校停水，厕所太臭睡不着。
突发灵感，想起之前在知乎上看到过的一篇文章：
https://www.zhihu.com/question/383684831/answer/2012191956558709772 

>入职第一天有什么瞬间让你马上想离职的？
>
>程序员，曾去一家公司，用git把代码拉下来，好奇看了眼git的提交纪录。
>
>晚上十一二点，甚至凌晨还在提交代码的情况比比皆是。
>
>跑路......


这个工具就是为了这个回答而生的——只需运行一个脚本，就能得到你所在仓库的提交时间统计，直观地看到团队的工作节奏。

## 功能

- 读取 Git 仓库的所有提交记录
- 提取每次提交的小时（0–23，基于提交者本地时区）
- 计算：
  - 总提交数
  - 简单平均小时
  - 圆形平均小时（解决跨午夜问题）
  - 中位数小时
  - 最早 / 最晚提交小时
- 可选绘制提交时间分布直方图（需要 `matplotlib`）

## 环境要求

- Python 3.6 或更高版本
- Git（当然，脚本需要在 Git 仓库中运行）

### 可选依赖

- `matplotlib`：用于绘制分布图  
  安装命令：`pip install matplotlib`

## 使用方法

1. 将脚本 `git_commit_time_stats.py` 保存到本地。
2. 打开终端，进入你想要分析的 Git 仓库目录。
3. 运行脚本：
   ```bash
   python git_commit_time_stats.py
